## 1 概述

RVDon PF Extension 已完成功能验证（rtlsim 7/7 PASSED, 红队独立验证通过, 28nm 综合报告完成）。本文档定义 RVDon 的**双线产品战略**和内存子系统 IP 授权策略。

### 1.1 双线战略

DiVo 的产品路径分两线，各有明确使命：

| | **A线：PF-1 FPGA 卡** | **B线：RVDon ASIC 卡** |
|---|---|---|
| **使命** | 最低成本验证 + 学术市场突破 | 丐版通用独立显卡市场 |
| **芯片** | Xilinx Kintex-7 KC160T (¥800) | 28nm/12nm ASIC |
| **GPU 能力** | ❌ 无（仅 PF 加速核心） | ✅ 完整 Vortex GPU（Vulkan/OpenGL） |
| **打游戏** | ❌ 不是产品目标 | ✅ "白天 Protenix，晚上 Minecraft" |
| **DDR4 直连** | ✅ 1× UDIMM 插槽 | ✅ 2× UDIMM/RDIMM 插槽 |
| **PCIe** | 3.0 x4 | 4.0 x8 (28nm) / 5.0 x8 (12nm) |
| **价格** | ¥3,999-5,999 | ¥2,000-3,000 (28nm) / ¥5,000-8,000 (12nm) |
| **产量** | 5-100 片 | 1,000-10,000 片 |
| **时间** | 立即可做 | NRE + 12-18 个月 |
| **做谁做** | DiVo 自建（KiCad + 嘉立创代工） | IP 授权客户 / 战略合作伙伴 |

### 1.2 核心设计理念

1. **带宽是伪问题，容量是唯一硬约束** — 用户自插 DDR4 内存条 = 零库存风险 + 垃圾佬友好
2. **FPGA 卡是造信用的工具，ASIC 卡才是造钱的产品** — 两线不冲突，互为前提
3. **ASIC 卡必须能打游戏** — 抢占丐版通用独立显卡市场，学生友好
4. **IP 授权是终局，但到达终局前需要一张入场券** — PF-1 就是那张入场券
5. **DiVo 永远不造芯片、永远不卖芯片** — ASIC 卡由授权客户制造和销售，DiVo 只提供 IP

### 1.3 架构选择原则

**A线 FPGA 卡只放 PF 核心，不放完整 Vortex：**

```
完整 Vortex GPU:  65,484 行 RTL → VU9P (350万 LUT, ¥50,000+) ← 不可行
PF 扩展核心:     ~2,700 行 RTL → Kintex-7 (10万 LUT, ¥800)   ← 可行

宿主机 CPU = Vortex 的 core/（取指/解码/调度/缓存）
FPGA        = Vortex 的 TCU PF 部分（三角乘/Flash Attention/距离掩码）
```

这不是退而求其次，而是正确的分工——PCIe 3.0 x4 带宽是 MD 计算需求的 103×，
瓶颈永远在计算不在传输。

**B线 ASIC 卡放完整 Vortex + PF 扩展：**

28nm ASIC 面积仅 ~4mm²（含 PF），成本 ¥540/颗，天然支持完整 GPU + PF。
学生白天跑 Protenix/MD，晚上打游戏——这是唯一能抢占丐版独立显卡市场的路径。

### 2.2 FPGA 资源利用

| 模块 | LUT 估算 | FF 估算 | BRAM | 占 KC160T |

| FA_SOFTMAX (含 32-entry LUT) | ~5,000 | ~4,000 | 4 | 5% |

| **剩余空间** | **79,000** | **183,000** | **75** | **78%** |

78% 剩余空间可用于：更多 PF 实例（并行度）、PF_TMM_CUTOFF 距离比较器、特殊函数单元。

### 2.3 PCIe 带宽分析

| 接口 | 带宽 | 延迟 | 100K 原子@?步/秒 | DDR4 直连 |
|------|:---:|:---:|:---:|:---:|
| PCIe 3.0 x4 (PF-1) | 3.9 GB/s | ~1 μs | 1,600 步/秒 | ✅ |
| PCIe 4.0 x8 (ASIC) | 15.75 GB/s | ~1 μs | 6,500 步/秒 | ✅ |
| USB 3.2 Gen2 | 1.2 GB/s | ~50 μs | 500 步/秒 | ❌ 无直连 |

**PCIe 3.0 x4 足够**：MD 计算瓶颈在 PF 核心算力（~32 步/秒 @ 100K 原子），不在带宽（3,292 步/秒支撑）。
带宽裕量 = 103×，即使 PCIe x1 都绰绰有余。

**USB 不可行**：①无 DDR4 直连（PF_TMM 需要本地访问坐标矩阵）；②延迟 50μs vs PCIe 1μs。

## 3 B线：RVDon ASIC 卡

**B线是 DiVo 的终局产品——"白天跑 Protenix，晚上打游戏"的丐版通用独立显卡。**

核心要点：
- **完整 Vortex GPU + PF 扩展** — 图形渲染（Vulkan/Mesa）+ 科学计算一体
- **游戏能力是刚需** — 抢占学生/个人开发者市场，这是与 FPGA 卡的本质区别
- **由 IP 授权客户制造和销售** — DiVo 不造芯片，但客户知道哪些需求是必要的

### 3.3 为什么用 DIMM 插槽而不是焊接内存？

| 理由 | 说明 |
|------|------|
| 用户自主 | 用户自选容量/价格平衡——垃圾佬插二手 16GB，企业插 128GB ECC |
| 零库存风险 | 卡厂不备内存库存，裸卡出货，像主板一样 |
| 独特市场定位 | 市面上没有任何加速卡让用户自插内存——对比 NVIDIA (焊接 HBM)、国产 GPU (焊接 GDDR) |
| 即插即用 | SPD 自动检测，零配置 |

## 4 内存子系统 IP 产品线

### 4.1 ARM Cortex 策略

ARM 将 Cortex-A55 (入门)、A76 (高性能)、X1 (旗舰) 拆分为独立 IP 授权。DiVo 将 DDR4/DDR5 控制器按内存类型拆分：

| IP 产品 | 代号 | 支持内存类型 | 门数 (28nm) | 定位 |
|---------|------|:---:|:---:|------|
| **RVDon-MC-U4** | Street | DDR4 UDIMM | ~15K | 垃圾佬丐版 |
| **RVDon-MC-R4** | Server | DDR4 UDIMM + RDIMM (自动检测) | ~30K | 专业版 |
| **RVDon-MC-L4** | Datacenter | DDR4 UDIMM + RDIMM + LR-DIMM | ~45K | 企业版 |
| **RVDon-MC-U5** | Street5 | DDR5 UDIMM | ~20K | 12nm 丐版 |
| **RVDon-MC-R5** | Server5 | DDR5 UDIMM + RDIMM | ~40K | 12nm 专业版 |
| **RVDon-MC-DUAL** | Flex | DDR4/DDR5 全双模 | ~50K | 12nm 旗舰 |

### 4.2 技术实现策略

MC-R4 是完整设计，MC-U4 是条件编译子集（与 PF 扩展 `VX_CFG_*_ENABLE` 模式一致）：

```systemverilog
module rvdon_mc import rvdon_mc_pkg::*; #(
    parameter VX_CFG_MC_RDIMM_ENABLE  = 1,  // MC-U4=0, MC-R4=1
    parameter VX_CFG_MC_LRDIMM_ENABLE = 0,  // MC-L4=1
    parameter VX_CFG_MC_DDR5_ENABLE   = 0   // MC-DUAL=1
) (
    // DFI interface to PHY
    // AXI interface to Vortex memory subsystem
    // I2C for SPD access
    // ...
);
```

### 4.3 PHY IP 归属

DDR4/DDR5 PHY 是 Synopsys/Cadence 第三方 IP，客户自行采购。DiVo 提供：
- **PHY 选型指南** — 哪个 Synopsys part number 匹配哪个 RVDon-MC
- **PHY-Controller 集成参考** — DFI 接口 RTL wrapper
- **已验证的 PHY 配置** — 时序参数和引脚分配

## 5 UDIMM + RDIMM 同插槽技术分析

### 5.1 结论：同一个 288-pin 插槽可以插 UDIMM 和 RDIMM

| 对比 | UDIMM | RDIMM |
|------|-------|-------|
| 引脚定义 | 288-pin (JEDEC 标准) | 288-pin (相同) |
| 注册时钟驱动 (RCD) | 无 | 有 (地址/命令/控制线) |
| SPD byte 3 | 0x01 | 0x02 |
| ECC | 可选奇偶校验 | 72-bit SECDED |
| 单条最大容量 | 32 GB | 128 GB |

### 5.2 自动检测流程

```
Boot:
  1. DDR PHY 上电
  2. I2C 读取 SPD EEPROM
  3. 解析 SPD byte 3:
     0x01 = UDIMM → 控制器配置为 UDIMM 模式
     0x02 = RDIMM → 控制器配置为 RDIMM 模式
     0x03 = LR-DIMM → 控制器配置为 LR-DIMM 模式
  4. RDIMM: 通过 MRS 命令初始化 RCD 寄存器
  5. 执行 JEDEC DRAM 初始化序列
  6. 运行 memory training (read/write leveling)
  7. RDIMM: 启用 ECC 引擎
  8. 标记内存就绪
```

### 5.3 限制

- **同一 channel 内不能混插 UDIMM 和 RDIMM**（电气负载和时序特性不同）
- **固件检测到混插时报错**（LED 指示或驱动错误码），拒绝启动内存子系统

## 9 风险与对策

| 风险 | 严重度 | 对策 |
|------|:---:|------|
| DIMM 信号完整性 | 中 | DDR4-3200 已充分验证; 遵循 JEDEC 参考布局; TDR/VNA 验证 |
| UDIMM/RDIMM 混插误操作 | 低 | 固件自动检测 + 报错; LED 指示; 清晰文档 |
| 二手 DIMM 质量 | 中 | 固件开机自检; RDIMM 路径有 ECC; 用户接受二手风险 |
| 大容量 DIMM 机械应力 | 低 | 加固 DIMM 插槽; 全高卡支撑架; DIMM 高度规格 |
| 4 DIMM 功耗 | 低 | 4 RDIMM ~16W; PCIe 插槽 25W 预算内 |
| DDR4 市场过时 | 中 | 12nm 卡支持 DDR5; 二手 DDR4 供应 10+ 年; 越老越便宜 |

## 12 MD 场景下的内存带宽分析

### 12.1 MD vs Protenix 内存需求对比

| 维度 | Protenix (推理) | MD (模拟) | 影响 |
|------|:---:|:---:|:---:|
| 访问模式 | 顺序 (矩阵遍历) | **随机** (邻居列表) | MD 更苛刻 |
| 容量需求 | 模型权重 64GB+ | 坐标+力+邻居 | Protenix 更大 |
| 带宽需求 | ~0.007 GB/s | **0.1-10 GB/s** | MD 高 100-1000× |
| FP64 需求 | 不需要 | **累加器必须** | MD 独有 |
| 瓶颈 | 容量 | **带宽 + 访问模式** | 不同 |

### 12.2 MD 带宽估算 (100K 原子系统)

```
目标: 1 ns/day (dt=2fs → 500K步/天 → 5.8 步/秒)

传统 MD (邻居列表随机读取):
  坐标读取: 100K × 200邻居 × 3 × 8B = 480 MB/步 (随机)
  力写入:   100K × 3 × 8B = 2.4 MB/步
  带宽需求: ~480 MB × 5.8 = ~2.8 GB/s
  
  DDR4-3200 理论: 51.2 GB/s
  随机访问有效带宽: 10-20% → 5-10 GB/s
  有效裕量: 2-3× ← 勉强够用

PF_TMM_CUTOFF (结构化瓦片遍历):
  距离矩阵瓦片: 8×8 顺序读取 → 缓存友好
  距离掩码: 硬件自动跳过远距对 → 减少无效访问
  有效带宽: 60-80% → 30-40 GB/s
  有效裕量: 8-12× ← 宽裕
```

### 12.3 PF_TMM_CUTOFF 改变游戏规则

| 方法 | 访问模式 | 有效带宽 | DDR4-3200 裕量 |
|------|----------|:---:|:---:|
| 传统 MD (邻居列表) | 随机跳跃 | 5-10 GB/s | 2-3× |
| **PF_TMM_CUTOFF** | 瓦片式顺序遍历 + 距离掩码 | 30-40 GB/s | **8-12×** |

PF_TMM_CUTOFF 消除了邻居列表构建步骤，直接在距离矩阵上做瓦片遍历。
DDR4 对顺序访问的效率远高于随机访问——这正是 RVDon 架构的优势。

### 12.4 各产品线 MD 场景覆盖

| 产品 | 内存带宽 | MD 10K 原子 | MD 100K 原子 | MD 1M 原子 |
|------|:---:|:---:|:---:|:---:|
| 28nm Street (2×DDR4) | 51.2 GB/s | ✅ 绰绰有余 | ✅ PF_TMM_CUTOFF | ❌ 不足 |
| 28nm Pro (2×DDR4 RDIMM) | 51.2 GB/s | ✅ | ✅ | ❌ |
| 12nm Pro (4×DDR5) | 204.8 GB/s | ✅ | ✅ 充裕 | ✅ 可用 |
| 12nm 旗舰 (4×DDR5 RDIMM) | 204.8 GB/s | ✅ | ✅ | ✅ 充裕 |

### 12.5 FP64 累加对带宽的影响

MD 路线图 Step 2 要求 FP64 累加器（混精度：FP32 输入 → FP64 累加）。
对带宽的影响：

| 操作 | FP32 | FP64 混精度 | 带宽增量 |
|------|:---:|:---:|:---:|
| 力写入 | 4B/element | 8B/element | +100% |
| 坐标读取 | 4B/element | 8B/element (如需FP64坐标) | +100% |
| 总带宽影响 | — | — | **+30-50%** |

DDR4-3200 双通道 51.2 GB/s → FP64 下有效 ~3-6 GB/s (传统 MD) / ~20-30 GB/s (PF_TMM_CUTOFF)。
仍然足够，但裕量进一步缩小——这强化了 12nm + DDR5 对 MD 的价值。
